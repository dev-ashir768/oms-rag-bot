========================================= SECTION 1: GENERAL SETTINGS

Picklist & Print Settings

Picklist Creation:
Picklist is the list of items a rider needs to collect—choose "By Orders" to pick per customer or "By Products" to pick the same items in bulk across orders.

By Orders:

Shows products grouped under each order.

You can see the Order ID next to each product.

Useful when the rider picks items order by order.

By Products:

Shows products grouped together across all orders, without listing Order IDs.

Useful when the rider picks all units of a product in bulk, minimizing walking in the warehouse.

AWB (AirWayBill) Details:

Include product details -> Shows all items on the label.

Simple -> Only basic info, no item details.

Show SKU -> Toggle SKU display on print.

AWB (AirWayBill) Layout:

Compact -> Small label.

Thermal -> Optimized for thermal printers.

Full page (A4) -> Prints on A4 paper.

Prints per page -> Choose 1, 2, or 3 AWBs per page.

WhatsApp Settings

Service Selection:

Official API -> Full interaction, cost per conversation.

Web API -> Simple notifications, cost per message.

Phone Number Field -> Add your WhatsApp number.

Actions:

QR Code -> Generate to connect your WhatsApp.

Chat Icon -> Link WhatsApp after scanning QR.

Search Icon -> Check connection status.

Trash Icon -> Remove linked number.

Scan & Update Settings

Select Status -> Pick a shipment stage (e.g., Booked, Ready for Return).

Assign Tags -> Enter tags that auto-apply when a shipment reaches this status.

How it works: Scan a shipment in scan and update module → update status → tags are automatically added.

========================================= SECTION 2: Orio Order Management System DASHBOARD

Total Outstanding:
The total amount still owed to you across all orders (regardless of date range). This includes all unpaid or partially paid orders up to the current date.

Current Outstanding:
The outstanding (unsettled) amount specifically for orders within the selected date range.

Estimated Available for Withdrawal:
The amount you are expected to receive from the courier/3PL partner after they collect payments from customers (applicable for Cash on Delivery orders). This is the net amount available for payout to you.

Orders:
Total number of orders placed within the selected date range.

Revenue:
Total order value (sum of all order amounts, including COD and prepaid) for orders in the selected date range.

Products Sold:
Total quantity of individual products/items sold within the selected date range.

Pending Payments by Shipping Partners:
The total order amount and shipment count that have been delivered by the courier but the payment has not yet been settled/transferred to you.

Order Flow Timeline:
Day-wise breakdown of courier tracking statuses (e.g., Picked Up, In Transit, Out for Delivery, Delivered, RTO, etc.) for shipments within the selected date range.

Order Status Trends:
Distribution of orders/shipments by status (e.g., Processing, Shipped, Delivered, Cancelled, Returned) showing both count and total order value for each status in the selected date range.

Revenue by Product:
Breakdown of revenue generated by each product, highlighting the best-selling and top-performing products in the selected date range.

Customer Distribution by Location:
Distribution of top customers and city‑wise distribution within the selected date range.

3PL Status:
Performance overview of third-party logistics (courier) partners:

Total shipments handled by each courier

Total order value associated with each courier

Breakdown by selected date range

Note: All metrics are dynamically updated based on the selected date range unless specified otherwise.

There are total three types of data table views the below description is about V1 order page data table. To change the view of the tables the buttons are on the top right hand side of the which on clicking reroutes to v1, v2 and v3
Data Table:
A comprehensive, sortable data table within the selected data range displaying detailed order information with the following columns and on top right hand side buttons are present like search bar, apply filter, calculate fare, fetch order, export, copy shipments and column visibility:

Table Structure:
Checkbox Column - Bulk selection for performing actions on multiple orders simultaneously

Murtaza Najmi Query Order Check Box Selected Orders for V1:

Change Status Dropdown on the left - A dropdown menu designed to update the operational status of multiple selected orders simultaneously. This feature includes a search bar for quick status retrieval. The available status options are:
New - Mark orders as newly received
Not Answered - Flag orders where customer didn't respond
First Call Attempt - Mark first contact attempt made
Second Call Attempt - Mark second contact attempt made
Third Call Attempt - Mark third contact attempt made Additional statuses available via search field

Perform Action DropDown on the right- A comprehensive command menu providing bulk operations for efficient order management. The available actions include:
Bulk Invoice - Automatically generates and downloads invoices for all selected orders.
Call Selected Orders - Initiates the calling workflow or queues the selected orders for the support team.
Push to WMS - Synchronizes the selected orders with the Warehouse Management System for inventory allocation and shipping.
Create CN - Generates Consignment Numbers (CN) for the selected orders to prepare them for logistics handover.
Create Load Sheet - Compiles the selected orders into a consolidated load sheet for courier dispatch.
Bulk Tracking - Triggers a real-time status update for the shipments associated with the selected orders.
Bulk 3PL Print - Batches and prints shipping labels using the integrated Third-Party Logistics provider's format.
Bulk OMS Print - Batches and prints shipping labels using the internal Order Management System's standard format.
Merge Orders - Combines multiple orders from the same customer into a single shipment to optimize shipping costs.
Add Bulk Tags - Applies specific identification tags (e.g., 'Priority', 'Fragile') to all selected orders for easier categorization.
Split Orders - Divides a single order into multiple sub-orders or shipments, useful for partial fulfillment.
Create Picklist - Generates a warehouse picklist specifically for the items contained within the selected orders.
Delete - Permanently removes or archives the selected orders from the system.

Shopify:

1. How to sync product inventory from OMS to shopify?
Answer:
First, integrate your Shopify store with the OMS. After that, go to the integration page where you will see your locations listed in the second row. Map these locations with your Shopify locations accordingly.
If you have more than one location in Shopify, you should create an additional location in the OMS and map all Shopify locations with the corresponding OMS locations.
Next, go to the Products tab. At the top, you will find the option for bulk product import. From there, select your Shopify store and import your products. Once imported, your products will automatically appear in the system and will be synced.
After this, you can go to the Inventory tab to view your products and their variations location-wise. You can also update inventory from there if needed.
If your products or inventory are synced from Shopify, any updates made in the OMS will also be reflected in Shopify. Similarly, any inventory changes made in Shopify will automatically update in the OMS.
This completes the two-way inventory synchronization between Shopify and the OMS.
